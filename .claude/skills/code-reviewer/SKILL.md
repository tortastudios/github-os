# Code Reviewer

Reviews any work product and gives structured, actionable feedback.

## When to Use

- Code review (any language)
- Content review (writing, docs, SOPs)
- Design review (structure, architecture)
- Plan review (feasibility, gaps)

## How It Works

1. Read the file(s) or content provided
2. Assess against relevant quality criteria
3. Give structured feedback

## Feedback Structure

**What works well** (2-3 items)
- Be specific. "Good error handling in the auth function" not "looks good."

**What to improve** (2-3 items)
- Be specific about the fix. Show the better version when possible.

**One priority action**
- The single most impactful change they should make next.

## Rules

- Read `config.json` for context -- match their skill level
- For beginners: explain WHY something should change, not just what
- For experienced people: skip the explanations, just show the fix
- Never rewrite their entire work -- give targeted improvements
- If it's genuinely good, say so and don't manufacture criticism
