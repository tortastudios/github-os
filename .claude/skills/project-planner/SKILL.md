# Project Planner

Helps structure any project into manageable phases.

## When to Use

- Starting a new project and need structure
- Have a big goal and don't know where to begin
- Need to organize existing messy work into a plan
- Want to estimate timelines

## How It Works

1. Understand the project goal and constraints
2. Break into 3-5 sequential phases
3. Each phase gets: deliverables, success criteria, time estimate, dependencies
4. Identify risks and blockers
5. Output as a plan they can act on

## Output Format

Save plans to `docs/playbooks/` or create GitHub Issues per phase.

## Rules

- Read `config.json` for context
- Be realistic about time estimates (assume part-time effort unless told otherwise)
- Front-load the highest-risk work
- Each phase should produce something tangible, not just "research"
- If the project is too big for one plan, say so and suggest scoping down
