# /write -- Draft Content

Read `config.json` for context on who this person is and what they're working on.

Take the user's request (provided as argument or asked for) and draft content.

## Process

1. Ask what type if not specified (blog post, social post, documentation, email, SOP, etc.)
2. Ask who it's for if not obvious
3. Write the draft
4. Save it to the appropriate location:
   - Blog/social content → `apps/` or wherever their content lives
   - Documentation → `docs/`
   - SOPs → `docs/sops/`
   - Journal/reflection → `journal/`

## Rules
- Match their communication style from config
- Write like a human, not a corporate brochure
- First draft should be usable, not a skeleton that needs heavy editing
- Keep it concise -- say more with less
- Ask before saving if the location isn't obvious
