# /review -- Get Feedback

Read `config.json` for context on who this person is and what they're working on.

Review whatever the user shares (file, idea, plan, code, writing) and give structured feedback.

## Output Format

**3 things that work well**
- Be specific about why

**3 things to improve**
- Be specific about how

**Suggested next step**
- One concrete action they can take right now

## Rules
- Match their communication style from config
- Be honest, not encouraging for the sake of it
- If something is bad, say so clearly but constructively
- If they share a file path, read the file first
- If they paste content, review what they pasted
- Don't rewrite their work unless asked -- just give feedback
