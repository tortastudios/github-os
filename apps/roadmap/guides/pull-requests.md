# Collaborate with Pull Requests

Pull Requests are how changes get reviewed before merging into main. Even solo, they create a record of what changed and why. For teams, they're essential. No one changes the main branch without review.

## How to do it

1. Create a branch and make your changes on it
2. Push the branch to GitHub
3. Go to your repo and click **Compare & pull request** (GitHub shows this automatically)
4. Write a title and description explaining what you changed and why
5. Click **Create pull request**
6. Review the changes, then click **Merge pull request** when satisfied

## Try it

```
From your experiment branch, open a Pull Request. Write a description of what you changed and why. Merge it when you're satisfied.
```

## Tips

- Even if you're working solo, PRs create a useful history of what changed and why.
- Keep PRs small and focused. One change per PR is easier to review than a grab bag of updates.
