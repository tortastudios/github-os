# Get Feedback on Your Work

Say "Review [filename] and give me feedback" to your AI tool (or run `/review` in Claude Code). It gives you structured feedback on anything -- a document, a plan, a piece of writing, code. It returns 3 strengths, 3 improvements, and a suggested next step. Use it before you share or ship anything.

## How to do it

1. Open the repo in your AI tool
2. Pick something you've written (a journal entry, SOP, decision log, or plan)
3. Say "Review [filename] and give me feedback" (or run `/review [filename]` in Claude Code)
4. Read the feedback: strengths, improvements, and next step
5. Make changes based on the feedback and commit

## Try it

```
Pick something you've written in your repo and say:
"Review journal/week-1.md and give me feedback"
(or run /review journal/week-1.md in Claude Code)
```

## Tips

- Review works on any text file in your repo. Use it on SOPs, decisions, plans, or writing drafts.
- The feedback is more useful when the file has enough content to evaluate. A single sentence won't give much to work with.
