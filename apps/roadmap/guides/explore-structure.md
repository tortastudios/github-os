# Explore the Repo Structure

Before changing anything, look around. Understanding what each folder is for makes everything else easier. apps/ is for projects, docs/ is for business knowledge, workflows/ is for processes, journal/ is for notes.

## How to do it

1. Open your repo on GitHub.com (or in your file explorer if you cloned it)
2. Click into each top-level folder
3. Read the README.md inside each one
4. Notice the .claude/ folder -- that's where your AI configuration lives
5. Look at config.json in the root -- that's your personal settings file

## Try it

```
Open your repo and click through every folder. Read each README.md file inside them. Pay special attention to config.json, agents.md, and the .claude/ folder.
```

- You don't need to understand everything right away. Just get a feel for where things live.
- The folder names are intentionally plain. If you're looking for something, it's probably where you'd guess.
