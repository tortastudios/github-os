# Document a Process You Repeat

If you do something more than twice, write it down. SOPs (Standard Operating Procedures) go in docs/sops/. Could be how you onboard a client, publish content, or handle support requests. Your AI tool can draft one for you from a description.

## How to do it

1. Think of a process you repeat regularly
2. Say "Help me write an SOP for [your process]" to your AI tool (or run `/write an SOP for [your process]` in Claude Code)
3. Review the output and adjust it to match how you actually work
4. Save it to docs/sops/ with a descriptive filename
5. Commit

## Try it

```
Open your AI tool and say:
"Help me write an SOP for [something you do at least once a week]"
(or run /write in Claude Code)

Save the result to docs/sops/your-process-name.md
```

## Tips

- Write SOPs as if the reader has never done the task before. Your future self will thank you.
- Update SOPs when the process changes. A wrong SOP is worse than no SOP.
- Use checklists (`- [ ]`) so people can track progress as they follow the steps.
