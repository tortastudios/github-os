# Create a Workflow Template

Workflows are documented processes in workflows/. Your repo includes templates for content pipelines and project kickoffs. Create your own for any process you repeat: client onboarding, weekly review, content publishing, sales follow-up.

## How to do it

1. Browse workflows/templates/ to see the existing templates
2. Decide on a process you want to document
3. Use /write in your AI tool: "/write a workflow template for [your process]"
4. Save the result to workflows/templates/
5. Commit

## Try it

```
Use /write to draft a workflow:
/write a workflow template for [your repeatable process]

Save to workflows/templates/your-workflow-name.md
```

## Tips

- Workflows are different from SOPs. A workflow describes the full flow from start to finish with stages. An SOP is the step-by-step instructions for one part.
- Start with the processes that cause you the most pain when they go wrong.
