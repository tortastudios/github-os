# Write a README

The README is your repo's front door. It's the first thing anyone sees when they visit your project. A good README answers three questions: what is this, why does it exist, and how do I use it. It's written in Markdown -- a simple formatting language that GitHub renders nicely.

## How to do it

1. Open your repo's **README.md** file
2. Click the pencil icon to edit
3. Add a clear title with `# Project Name`
4. Write one sentence explaining what the project is
5. Add a **Getting Started** section with basic instructions
6. Commit your changes

## Try it

```markdown
# My Project

A short description of what this project does and who it's for.

## Getting Started

1. Clone this repo
2. Open the files
3. Start making changes

## What's Inside

- `README.md` -- this file
- `docs/` -- documentation and notes
```

## Tips

- Use headings (`##`), bullet points (`-`), and code blocks (triple backticks) to make your README scannable.
- You don't need to write everything at once. Start with the basics and improve it over time.
