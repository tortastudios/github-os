# Use Issues as a Task Tracker

GitHub Issues is a built-in task tracker. Each issue is a task, bug report, or idea. You can assign them to people, add labels, set milestones, and close them when done. It's simpler than most project management tools and it lives right next to your code.

## How to do it

1. Go to your repo and click the **Issues** tab
2. Click **New issue**
3. Write a clear title (one line describing the task)
4. Add details in the description -- what needs to happen and why
5. Add labels (click the gear icon next to Labels)
6. Click **Submit new issue**

## Try it

```
Create 3 issues in your repo for things you need to do this week:

Issue 1: "Set up project README with getting started instructions"
Issue 2: "Add .gitignore for common files"
Issue 3: "Research hosting options for the website"

Add the label "priority" to the most important one.
```

## Tips

- Close issues with commit messages by writing `Closes #1` in your commit. GitHub will automatically close issue number 1 when the commit hits the main branch.
- Use labels to categorize: `bug`, `feature`, `idea`, `priority`. Create whatever labels make sense for your workflow.
- Issues are numbered sequentially. You can reference them anywhere on GitHub with `#number`.
