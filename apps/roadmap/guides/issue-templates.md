# Customize Your Issue Templates

Your repo came with templates for ideas, tasks, and questions in .github/ISSUE_TEMPLATE/. Customize them for your actual workflow, or add new ones. If you onboard clients, make a "New Client" template. If you produce content, make a "Content Brief" template.

## How to do it

1. Open the .github/ISSUE_TEMPLATE/ folder in your repo
2. Pick an existing template to edit (idea.md, task.md, or question.md)
3. Change the sections and prompts to match your workflow
4. To add a new template, create a new .md file in the same folder
5. Commit your changes

## Try it

```
Open .github/ISSUE_TEMPLATE/ and edit one of the existing templates. Change the prompts to match how you actually work. Or create a new template for a workflow you use often.
```

## Tips

- Keep templates short. If they're too long, people skip the fields. Three to five sections is enough.
- Include a checklist so the person creating the issue can define "done" upfront.
