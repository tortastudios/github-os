# Track a Task with GitHub Issues

GitHub Issues is your built-in task tracker. Each issue is a task, idea, or question with its own discussion thread. Your repo comes with templates for ideas, tasks, and questions so you don't start from a blank page. Better than sticky notes, cheaper than project management software.

## How to do it

1. Go to your repo's **Issues** tab
2. Click **New issue**
3. Pick a template -- Task, Idea, or Question
4. Fill in the fields the template gives you
5. Click **Submit new issue**

## Try it

```
Go to your repo's Issues tab, click "New issue", pick the "Task" template, and create an issue for something you need to do this week.
```

## Tips

- Close issues with commit messages by writing `Closes #1` in your commit. GitHub automatically closes the issue when the commit hits main.
- Issues are numbered sequentially. Reference them anywhere on GitHub with `#number`.
