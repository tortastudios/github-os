# Log a Decision

Six months from now you'll ask "why did I do it this way?" The decision log answers that. Your repo includes a template at docs/decisions/000-template.md. Copy it, fill it out for a real decision, and commit. Context, decision, reasoning, consequences -- all captured.

## How to do it

1. Open docs/decisions/000-template.md
2. Copy it to a new file like docs/decisions/001-chose-github-os.md
3. Fill in each section: date, status, context, decision, consequences
4. Commit the new file

## Try it

```
Copy docs/decisions/000-template.md to docs/decisions/001-chose-github-os.md. Fill it out: why did you decide to use GitHub OS? What alternatives did you consider?
```

## Tips

- Don't overthink it. Even a two-sentence entry is better than nothing.
- Record decisions when you make them, not weeks later when you've forgotten the reasoning.
- Number decisions sequentially so they're easy to reference ("see decision 003").
