# Project Kickoff

What to do when starting something new.

## Before You Build

- [ ] Can you explain what it does in one sentence?
- [ ] Who is it for?
- [ ] What does "done" look like?
- [ ] What's the simplest version you could ship?

## Setup

- [ ] Create folder in `apps/[project-name]/`
- [ ] Add a README with the one-sentence description
- [ ] Create a GitHub Issue for the first milestone
- [ ] Run `/plan` to break it into phases

## First Milestone

Ship something visible within the first week. A landing page, a working prototype, a demo -- anything real.
