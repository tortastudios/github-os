# Content Pipeline

A simple process for going from idea to published content.

## Steps

1. **Capture** -- Write the idea in a GitHub Issue (use the "idea" template)
2. **Draft** -- Use `/write` to create a first draft
3. **Review** -- Use `/review` to get feedback
4. **Edit** -- Revise based on feedback
5. **Publish** -- Use `/ship` to deploy or post
6. **Close** -- Close the GitHub Issue

## Where Content Lives

- Blog posts: `apps/[your-site]/blog/`
- Social posts: wherever you manage social
- Documentation: `docs/`

## Cadence

Set your own pace. Consistency matters more than volume.
